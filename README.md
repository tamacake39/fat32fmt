# fat32fmt

Windows 環境において、物理ドライブに対して直接 FAT32 フォーマットを強制執行するコマンドラインツールです。OS の標準フォーマットでは制限される 32GB を超える大容量ドライブに対しても、FAT32 構成を適用します。

## fat32fmt 技術仕様

本ツールは、Windows における FAT32 の 32GB 制限を回避し、物理セクタへのダイレクト I/O（DASD）を用いて FAT32 ボリュームを強制構築します。

### 1. 物理・論理ディスクレイアウト

ディスクの先頭領域は以下の固定オフセットで構成されます。

| 項目                     | 設定値 / オフセット        | 備考                              |
| :----------------------- | :------------------------- | :-------------------------------- |
| **パーティション形式**   | GPT (GUID Partition Table) | `PARTITION_STYLE_GPT`             |
| **データ開始位置 (LBA)** | `2048` (1MB)               | `PARTITION_START_LBA`             |
| **セクタサイズ**         | `512` Bytes                | `SECTOR_SIZE`                     |
| **予約セクタ数**         | `32`                       | VBR から FAT 開始までのオフセット |
| **FAT 数**               | `2`                        | FAT1 および FAT2                  |

### 2. ファイルシステム・パラメータ (FAT32)

アロケーションおよびアドレッシングに関する固定仕様です。

-   **アロケーションユニットサイズ (Cluster Size)**: **32 KB 固定**
    -   `sectors_per_cluster = 64` (32KB / 512B)
-   **FAT テーブル領域**:
    -   サイズ：ディスク容量に基づき動的に計算
    -   初期化範囲：`sectors_per_fat` 全域を `0x00` で埋め、先頭 12 バイトのみメディア型等のシグネチャを書き込み
-   **ルートディレクトリ**:
    -   開始位置：`Cluster 2`
    -   初期化：最初の 1 クラスタ（32KB）を完全にゼロクリア

### 3. 実装上のクリティカル・パス

#### 3.1 ボリューム・デタッチ・シーケンス

物理書き込み時の OS によるキャッシュ干渉（Delayed Write）を防ぐため、以下の順序で排他権限を取得します。

1.  `FSCTL_ALLOW_EXTENDED_DASD_IO`: 境界チェックを緩和し、パーティション外へのアクセスを許可。
2.  `FSCTL_LOCK_VOLUME`: ファイルシステムレベルのロック。
3.  `FSCTL_DISMOUNT_VOLUME`: メモリ上の全バッファを強制破棄。
4.  `IOCTL_DISK_SET_DISK_ATTRIBUTES`: `DiskAttributeOffline` をセットし、エクスプローラーの再スキャンを抑止。

#### 3.2 物理整合性の確保

-   **強制同期**: `WriteFile` 実行直後に `FlushFileBuffers` を発行し、ハードウェアの書き込みキューを空にします。
-   **冗長 VBR**: セクタ `+6` (Backup Boot Sector) に対して、メインの VBR (LBA 2048) と同一の内容を書き込みます。
-   **GPT GUID**: パーティションタイプには `PARTITION_BASIC_DATA_GUID` (`EBD0A0A2-...`) を使用。

### 4. コンパイル・環境依存性

-   **ビルドターゲット**: x86/x64 Windows (Native API / Win32 API)
-   **リンクライブラリ**: `shell32.lib` (`IsUserAnAdmin` 用)
-   **主要マクロ**:
    -   `#pragma pack(1)` を使用し、VBR および FSINFO 構造体のアライメントを厳密に制御。
    -   `IOCTL_DISK_SET_DISK_ATTRIBUTES` が未定義の環境（古い MinGW 等）向けに、`0x0007c0f0` を直接定義。

### 3. 安全機能（強化済み）

-   **システムドライブ保護**: `IOCTL_STORAGE_GET_DEVICE_NUMBER` を使用し、現在 Windows が稼働している物理ディスク番号を特定して保護します。
-   **排他ロック**: `FSCTL_LOCK_VOLUME` および `FSCTL_DISMOUNT_VOLUME` を発行し、OS やエクスプローラーによる干渉を完全に排除した状態で書き込みます。

## コンパイル方法

本ツールは Win32 API および Common Controls を使用します。以下の手順でビルドすることで、管理者昇格マニフェストを含んだ単一の実行ファイルが生成されます。

### MinGW (gcc) の場合

1. **リソースのコンパイル** (マニフェストの埋め込み)

    ```bash
    windres resource.rc -o resource.o
    ```

2. **本体のビルドとリンク**
    ```bash
    gcc main.c resource.o -o fat32fmt.exe -Wall -luser32 -lgdi32 -lcomctl32 -lshell32 -lshlwapi -s
    ```

### リンクライブラリ解説

-   `-luser32 / -lgdi32`: GUI ウィンドウおよび描画用
-   `-lcomctl32`: モダンな UI コントロール（コンボボックス等）用
-   `-lshell32`: 権限チェック用
-   `-lshlwapi`: システムパス・デバイス特定用

## 使用方法

### GUI モード（推奨）

エクスプローラーから `fat32fmt.exe` をダブルクリックして起動します。
直感的な操作でドライブを選択し、安全にフォーマットを実行できます。

### CUI モード

コマンドプロンプト等から引数を指定して実行します。

| コマンド                  | 内容                                         |
| :------------------------ | :------------------------------------------- |
| `fat32fmt.exe -l`         | 利用可能な物理ドライブ一覧を表示             |
| `fat32fmt.exe -f [index]` | 指定したインデックスのドライブをフォーマット |
| `fat32fmt.exe -h`         | ヘルプを表示                                 |
